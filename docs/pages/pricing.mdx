# Pricing & Access

Each API has its own pricing model and access requirements. This overview helps you understand the costs and how to get started.

## CourtListener API

### Free Tier
- **125 requests/day** with default limits
- Access to all endpoints
- Community support

### Commercial Access
Free Law Project memberships and commercial agreements available for expanded access. Contact [Free Law Project](https://www.courtlistener.com/contact/) for details.

## Caselaw Access Project API

### Free Tier
- **500 requests/day**
- Access to 6.7 million cases
- Full case text and metadata

### Research Access
Higher limits available for research projects. Contact [Harvard Library Innovation Lab](https://lil.law.harvard.edu/) for research partnerships.

## Congress.gov API

### Free Access
- **5,000 requests/hour**
- No cost for public data
- Official government API

### API Key
Required but free. Sign up at [api.congress.gov/sign-up](https://api.congress.gov/sign-up/).

## Vaquill AI API

### Free Tier
- **500 credits/month**
- No enterprise contract required
- Access to all endpoints

### Credit Costs
Different endpoints consume different credits:
- **AI Legal Q&A**: 5-10 credits per query
- **Case Search**: 3-5 credits per search
- **Statute Lookup**: 1-2 credits per lookup
- **Contract Analysis**: 10-20 credits per analysis

See [vaquill.ai/pricing](https://vaquill.ai/pricing) for current rates.

## Getting Access

### Quick Start
1. **CourtListener**: Create account at [courtlistener.com](https://www.courtlistener.com/account/)
2. **CAP**: Sign up at [case.law](https://case.law/user/details/)
3. **Congress.gov**: Register at [api.congress.gov/sign-up](https://api.congress.gov/sign-up/)
4. **Vaquill AI**: Get API key at [app.vaquill.ai](https://app.vaquill.ai)

### Enterprise Solutions
For high-volume or commercial use cases, consider:
- **CourtListener**: Commercial agreements with Free Law Project
- **CAP**: Research partnerships with Harvard
- **Vaquill AI**: Enterprise plans with custom limits

## Cost Optimization

### Tips to Reduce Costs
- Cache frequently accessed data
- Use specific queries to reduce result size
- Implement efficient pagination
- Choose the right API for your use case
- Monitor usage and optimize accordingly
