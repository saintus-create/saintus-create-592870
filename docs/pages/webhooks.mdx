# Webhooks

Subscribe to CourtListener events and receive real-time notifications when new data is added or updated.

## Overview

Webhooks allow you to receive HTTP POST notifications when specific events occur in the CourtListener system, enabling real-time data synchronization and automated workflows.

## Supported Events

### Opinion Events
- `opinion.created` - New opinion added
- `opinion.updated` - Opinion metadata updated
- `opinion.deleted` - Opinion removed

### Docket Events
- `docket.created` - New docket created
- `docket.updated` - Docket information updated
- `docket.deleted` - Docket removed

### RECAP Events
- `recap.document_added` - New RECAP document uploaded
- `recap.docket_updated` - RECAP docket updated

### Person Events
- `person.created` - New person added
- `person.updated` - Person information updated

## Setting Up Webhooks

### Register Webhook

```bash
curl -X POST "https://www.courtlistener.com/api/rest/v4/webhooks/" \
  -H "Authorization: Token YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-server.com/webhook",
    "events": ["opinion.created", "docket.updated"],
    "secret": "your-webhook-secret"
  }'
```

### Webhook Configuration

| Parameter | Type | Description |
|-----------|------|-------------|
| `url` | string | Your webhook endpoint URL |
| `events` | array | List of events to subscribe to |
| `secret` | string | Secret for signature verification |

## Security

### Signature Verification

Each webhook includes a signature header for verification:

```bash
X-Webhook-Signature: sha256=signature_hash
```

### Verify Signature

```python
import hmac
import hashlib

def verify_signature(payload, signature, secret):
    expected_signature = hmac.new(
        secret.encode(),
        payload.encode(),
        hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(expected_signature, signature)
```

### Best Practices
- Use HTTPS for webhook URLs
- Implement signature verification
- Validate webhook payloads
- Use secrets for each webhook

## Managing Webhooks

### List Webhooks

```bash
curl -X GET "https://www.courtlistener.com/api/rest/v4/webhooks/" \
  -H "Authorization: Token YOUR_API_KEY"
```

### Delete Webhook

```bash
curl -X DELETE "https://www.courtlistener.com/api/rest/v4/webhooks/{id}/" \
  -H "Authorization: Token YOUR_API_KEY"
```

### Update Webhook

```bash
curl -X PUT "https://www.courtlistener.com/api/rest/v4/webhooks/{id}/" \
  -H "Authorization: Token YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "events": ["opinion.created", "opinion.updated"]
  }'
```

## Limitations

- Maximum 10 webhooks per account
- Rate limits apply to webhook creation
- Webhook URLs must be publicly accessible
- Payload size limit: 1MB
